# 🚀 RouteC AsyncLogger (高性能异步日志库)

一个基于 C++11 实现的**工业级异步日志库**。采用生产者 - 消费者模型，支持多线程并发写入、自动日志轮转、毫秒级时间戳及单例模式。经基准测试，在强制落盘场景下，性能比同步日志提升 **1000 倍+**。

## ✨ 核心特性

- **⚡ 极致性能**：异步双缓冲队列设计，主线程无阻塞。实测 5000 条强制落盘日志，耗时从 16s 降至 16ms。
- **🔒 线程安全**：使用 `std::mutex` + `std::condition_variable` 实现高效同步，支持多线程并发写入不丢包、不乱序。
- **💾 自动轮转**：内置 Log Rotation 机制，当日志文件超过设定阈值（如 1MB）自动切割备份，防止磁盘写爆。
- **🛠️ 易用接口**：基于单例模式 (Singleton) 封装全局宏 `LOG_INFO("msg")`，调用简洁，零样板代码。
- **📅 结构化日志**：自动附加 `[YYYY-MM-DD HH:MM:SS.mmm] [LEVEL]` 格式，便于排查问题。

## 📊 性能基准测试 (Benchmark)

在 WSL2 (Ubuntu) 环境下，对比同步日志 (Sync) 与 异步日志 (Async) 写入 **5,000 条** 日志并强制落盘 (`fsync`) 的耗时：

| 模式 | 耗时 (ms) | 备注 |
| :--- | :--- | :--- |
| **同步日志 (Sync)** | ~16,800 ms | 每次写入都阻塞等待磁盘 IO |
| **异步日志 (Async)** | ~16 ms | 批量写入，主线程几乎无感知 |
| **性能提升** | **≈ 1000 倍** | 🚀🚀🚀 |

> 💡 **结论**：在高并发、高吞吐场景下，异步日志是保障系统稳定性的关键组件。

## 🏗️ 技术架构

- **设计模式**：
  - **Producer-Consumer**：主线程生产日志，后台线程消费写入。
  - **Singleton**：全局唯一实例，确保资源统一调度。
- **核心技术**：
  - `std::thread`：后台持久化线程。
  - `std::queue` + `swap`：双缓冲队列，减少锁竞争。
  - `std::condition_variable`：条件变量实现线程休眠/唤醒，CPU 占用率趋近于 0。
  - `RAII`：`std::lock_guard` 确保异常安全，资源自动释放。
